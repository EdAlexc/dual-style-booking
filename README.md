# Studio MUA — Next.js version

Next.js 15 (App Router) port of the Studio MUA site (the original TanStack Start / Vite app lives at the repo root). Layout and styling are identical; all design tokens live in [`src/styles.css`](src/styles.css) — colors, fonts, gradients, and component utilities can be edited there without touching components.

## Run locally / in a GitHub Codespace

```bash
cd nextJS_version
npm install
cp .env.example .env.local   # then fill in the two Supabase values (same as the root .env's VITE_SUPABASE_* vars)
npm run dev
```

Open http://localhost:3000. In a Codespace, port 3000 is forwarded automatically — open the "Ports" tab and follow the forwarded URL (the dev server already binds `0.0.0.0`).

The booking form works without env vars until submit; the final step needs the Supabase credentials.

## Deploy to Vercel

1. Import the repository on [vercel.com/new](https://vercel.com/new).
2. Set **Root Directory** to `nextJS_version` (framework preset Next.js is auto-detected).
3. Add the environment variables `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`.
4. Deploy.

## Project notes

- **Dual theme** (`glam` / `bold`) is driven by `data-theme` on `<html>`; persisted in `localStorage("mua-theme")` and applied before first paint by an inline script in [`src/app/layout.tsx`](src/app/layout.tsx).
- **Global styles & design tokens**: [`src/styles.css`](src/styles.css) (Tailwind CSS v4, CSS-first — no tailwind.config). The "Component design tokens" section at the bottom holds the gradients and display-font utilities used by the landing, work, and book pages.
- **Landing videos**: drop MP4s at `public/videos/glam.mp4` and `public/videos/bold.mp4`; the poster images show if they're missing.
- **Booking backend**: anonymous insert into the Supabase `bookings` table (schema in `../supabase/migrations/`).
- `/sitemap.xml` is generated by [`src/app/sitemap.ts`](src/app/sitemap.ts).
